Relacionales (SQL)
Se basan en tablas con un montón de columnas para los datos, y filas para los registros.
Cada fila, tiene una primary key. Se utiliza como identificador externo. Las bases de datos
los utilizan para crear relaciones con otras tablas.
Son torpes para la escabilidad.

No relacionales (NoSQL)
Ayudan a las consultas en tiempo real.
Base datos de documentos: Suelen ser JSON. No crea relaciones.
Base de datos de documentos relacionados: Permiten tener relaciones. Internamente, las crean
en tiempo de ejecución (Mongo).
Base datos key-value: Identificador único (Cassandra).
Grafos: Crean relaciones entre nodos. Son útiles para redes sociales, o análisis de datos complejos (Neo4J).

Configurar Mongo Atlas
1) Crear cuenta en https://www.mongodb.com/cloud/atlas
Es buena práctica limitar el acceso por IP.

Un cluster tiene tres instancias de MongoDB, por lo que se garantiza la alta disponibilidad.
Mongo no tiene schemas, pero es una mala práctica no crearlas, porque no se sabe qué tipo de información hay en las bases.
Con mongoose se crea los schemas de Mongo en el código.

Gestionar las conexiones
1) Tener una conexión que se abre y se cierra con cada petición. Sirve para pocas peticiones.
2) Para mucho tráfico, es buena práctica compartir la conexioń.
